# Monetization Playbook - stage-pilot

This file is a commercial planning artifact, not a guarantee of revenue. Use it to make the repository legible as a sellable product, pilot, workshop, license, or proof-of-value package.

## Commercial Lane

- **Lane:** B2B developer tooling
- **Primary buyer:** AI platform teams and developer-tool teams shipping agents that need robust tool-call parsing.
- **Revenue wedge:** Parser/runtime/benchmark surface that improves malformed tool-call recovery.

## Offer Ladder

1. **npm package support** - pricing hypothesis: OSS free + support
2. **Parser reliability audit** - pricing hypothesis: $3k-$12k audit
3. **Benchmark pack and integration help** - pricing hypothesis: $2k-$8k/month package support

## Acquisition Channels

- npm ecosystem content
- Benchmark blog posts
- Agent runtime demos

## Proof Gates Before Selling

- verify passes
- Published package surface is clear
- Benchmark claim is reproducible

## Risk Boundaries

- Provider-neutral fixtures only
- Customer prompts need evals
- No universal reliability guarantee

## 30/60/90 Day GTM

- **30 days:** Publish benchmark report
- **60 days:** Create integration guide
- **90 days:** Offer parser audit

## Metrics That Matter

- Tool-call recovery
- Mutation coverage
- Integration defects avoided

## Positioning Note

Lead with the specific operational pain, then show the runnable proof, then offer a narrow pilot. Do not pitch the repository as a vague app; pitch it as a concrete workflow with evidence, risk boundaries, and a next buying step.
